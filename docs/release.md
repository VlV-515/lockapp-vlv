# Release Workflow

LockApp-vlv public releases use GitHub Releases as the primary channel.

This project currently uses a free Apple Developer account. Release ZIPs are ad-hoc signed only, not Developer ID signed, and not notarized. Public copy must state that macOS Gatekeeper can show a warning.

## Version

Current release version:

```text
1.1.0
```

Bundle metadata is generated by `scripts/package-app.sh`:

- `CFBundleShortVersionString`: release version.
- `CFBundleVersion`: release version.
- `CFBundleIdentifier`: `dev.vlv.lockappvlv`.
- `LSMinimumSystemVersion`: `13.0`.

## Build Release Assets

```bash
swift build
./scripts/package-release.sh 1.1.0
```

Expected files:

```text
dist/LockApp-vlv-1.1.0-macos-unsigned.zip
dist/LockApp-vlv-1.1.0-macos-unsigned.zip.sha256
```

Verify checksum from `dist` so the `.sha256` file remains portable:

```bash
cd dist
shasum -a 256 -c LockApp-vlv-1.1.0-macos-unsigned.zip.sha256
```

## Publish GitHub Release

Use the committed source state that produced the ZIP.

```bash
git tag v1.1.0
git push origin main
git push origin v1.1.0
gh release create v1.1.0 \
  dist/LockApp-vlv-1.1.0-macos-unsigned.zip \
  dist/LockApp-vlv-1.1.0-macos-unsigned.zip.sha256 \
  --title "LockApp-vlv 1.1.0" \
  --notes-file .github/release-notes/v1.1.0.md
```

If the release already exists and the local assets are the intended files:

```bash
gh release upload v1.1.0 \
  dist/LockApp-vlv-1.1.0-macos-unsigned.zip \
  dist/LockApp-vlv-1.1.0-macos-unsigned.zip.sha256 \
  --clobber
```

Verify:

```bash
gh release view v1.1.0 --json assets,url,publishedAt,tagName,name
```

## Install Notice

Release notes and README must keep this warning:

```text
This build is ad-hoc signed only. It is not signed with an Apple Developer ID certificate and it is not notarized by Apple. macOS Gatekeeper may show a warning.
```

Do not claim Developer ID signing or notarization until the project has a paid Apple Developer Program account and a verified notarized build.
